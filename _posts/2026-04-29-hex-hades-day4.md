## 2. 복구 작업

### 🔧 진행 순서

```
1. 진단: 프로세스 상태 확인 → 모든 PID DEAD
2. 버그 수정: run_hades.py 인코딩 에러 (cp949 vs UTF-8)
3. 데이터 백필: 2일치 5분봉 수집 (BTC/ETH/SOL)
4. ML 서비스 재기동: LightGBM 모델 로드 확인
5. HADES 루프 테스트: 1회 사이클 정상 동작 확인
6. 백그라운드 루프 기동: 5분간격 자동 실행
```

### 🐛 발견한 버그

**run_hades.py 인코딩 에러:**
```python
# Before (Windows cp949에서 crash)
print(f"[HADES] {now} — 사이클 시작")  # — = U+2014 em-dash

# After
print(f"[HADES] {now} - cycle start")  # ASCII 대체
```

Windows 콘솔이 cp949를 사용하는데, 스크립트 내부에 Unicode em-dash가 있어 `UnicodeEncodeError`가 반복 발생하고 있었습니다.

> 💡 **교훈:** 크로스플랫폼 코드에서는 항상 ASCII 또는 명시적 UTF-8 출력을 보장해야 합니다. Python 3.12에서도 Windows 기본 인코딩은 여전히 cp949입니다.
